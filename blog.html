2. What are the differences between map, forEach, filter,and find?

Map

  • The map() method creates a new array populated with the results of calling a provided function on every element in the calling array.
  • Don’t modify old array

Foreach

  • The forEach() method executes a provided function once for each array element.
  • We don’t return anything from it. We just do something in it

Filter

  • The filter() method creates a shallow copy of a portion of a given array, filtered down to just the elements from the given array that pass the test implemented by the provided function.
  • Filter return all of them which are matched the condition

Find

  • The find() method returns the first element in the provided array that satisfies the provided testing function. If no values satisfy the testing function, undefined is returned.
  • Find return only the first match of the conditions

3. What are the differences between arrow function and regular function?

  • An arrow function expression is a compact alternative to a traditional function expression, but is limited and can't be used in all situations.
  • Arrow functions don't have their own bindings to this, arguments or super, and should not be used as methods. On the other hand traditional function expression can do all that.
  • Arrow functions don't have access to the new.target keyword.
  • Arrow functions aren't suitable for call, apply and bind methods, which generally rely on establishing a scope.
  • Arrow functions cannot be used as constructors.
  • Arrow functions cannot use yield, within its body.

Template literals are enclosed by backtick (`) characters instead of double or single quotes, allowing for the following, which are not possible in normar double or single quote.
